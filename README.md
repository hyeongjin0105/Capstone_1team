# 영남대학교 호주 글로벌 SW Capstone 1team 프로젝트

> **팀원을 위한 협업**
> 프로젝트의 화면 구조 설계부터 핵심 기능 구현까지의 개발 로드맵입니다.

---

## 화면 구조 설계 (Scene Architecture)
사용자 경험을 고려한 직관적인 화면 구조를 구성합니다.

*   **인증 시스템**: 로그인 및 회원가입 화면
*   **메인 대시보드**: 프로젝트 전반의 현황을 한눈에 확인하는 메인 화면
*   **워크스페이스**: 팀 생성 및 참여 관리 화면
*   **협업 도구**:
    *   할 일 목록(To-do List) 화면
    *   회의록 작성 및 보관 화면

---

## 프론트엔드 디자인 시스템 (UI/UX)
일관성 있는 디자인과 컴포넌트 재사용성을 극대화합니다.

*   **디자인 통일**: 글로벌 스타일 가이드 적용 (메뉴바, 버튼, 색상 팔레트, 글꼴 정리)
*   **레이아웃 표준화**: 공통 레이아웃 구조를 적용하여 각 페이지의 일관성 유지
*   **반응형 웹(Responsive)**: 모바일 및 태블릿 환경에서도 화면이 깨지지 않는 반응형 UI 구현

---

## 3. 할 일 관리 핵심 기능 (Task Management)
체계적인 업무 분담과 일정 관리를 위한 기능을 구현합니다.

*   **Task CRUD**: 할 일 등록, 수정, 삭제 기능
*   **업무 컨텍스트 설정**: 담당자 지정 및 마감일(Due Date) 설정
*   **상태 관리**: 할 일 완료 여부 및 진행 상황 변경
*   **필터링**: 팀별/프로젝트별 할 일 필터링 조회

---

## 4. 회의록 기능 (Meeting Minutes)
팀원 간의 의사소통 기록을 투명하게 관리합니다.

*   **기본 정보 기록**: 회의 날짜 및 참석자 명단 작성
*   **회의 요약**: 회의 주요 내용 및 핵심 결정 사항 기록
*   **액션 아이템**: 다음 회의 전까지 수행해야 할 팀별/개인별 할 일 연동 기록

---

## 5. 테스트 데이터 및 QA (Test Data & Validation)
기능 검증을 위한 샘플 데이터 셋업 및 최종 테스트 진행 예정입니다.

*   **시나리오 데이터 구성**:
    *   테스트용 사용자 데이터 (약 5명)
    *   테스트용 팀/워크스페이스 (2~3개)
    *   기능 확인용 샘플 할 일 및 회의록 데이터 입력
*   **기능 검증**: 구현된 기능별 정상 작동 여부 및 예외 처리 최종 확인
